# ZJU 课表助手 - 应用设计文档

## 应用概述

ZJU 课表助手是一个为浙江大学学生设计的移动应用，用于查看和管理个人课表。应用通过与浙江大学本科教学管理信息服务平台集成，为用户提供精致的课表展示和管理功能。

---

## 屏幕列表

### 1. **登录屏幕 (Login Screen)**
- **目的**: 用户身份验证
- **内容**: 
  - 应用logo和标题
  - 用户名输入框
  - 密码输入框
  - 登录按钮
  - 错误提示信息
- **功能**: 
  - 输入浙江大学账号凭证
  - 与CAS系统集成进行身份验证

### 2. **课表主屏幕 (Schedule Screen)**
- **目的**: 展示当前周课表
- **内容**:
  - 周次选择器（上周/下周导航）
  - 精致的课表网格
  - 课程卡片（显示课程名、教室、时间）
  - 当前日期指示器
- **功能**:
  - 按周浏览课表
  - 点击课程卡片查看详情
  - 课程颜色编码区分不同课程

### 3. **课程详情屏幕 (Course Detail Screen)**
- **目的**: 展示单个课程的详细信息
- **内容**:
  - 课程名称
  - 授课教师
  - 教室位置
  - 上课时间（周次、星期、节次）
  - 课程描述（如有）
- **功能**:
  - 返回课表
  - 可能的后续操作（如添加提醒）

### 4. **设置屏幕 (Settings Screen)**
- **目的**: 应用设置和用户管理
- **内容**:
  - 用户信息显示
  - 登出按钮
  - 应用设置选项
- **功能**:
  - 查看当前登录用户
  - 登出并返回登录屏幕

---

## 主要用户流程

### 流程 1: 初次使用 - 登录
1. 用户启动应用 → 显示登录屏幕
2. 用户输入浙江大学账号和密码
3. 点击登录按钮
4. 应用与CAS系统通信进行验证
5. 验证成功 → 跳转到课表主屏幕
6. 验证失败 → 显示错误提示，用户可重试

### 流程 2: 查看课表
1. 用户在课表主屏幕
2. 默认显示当前周课表
3. 用户可点击"上周"或"下周"按钮切换周次
4. 课表动态更新，显示对应周次的课程
5. 用户点击课程卡片 → 进入课程详情屏幕

### 流程 3: 查看课程详情
1. 用户从课表主屏幕点击课程卡片
2. 进入课程详情屏幕
3. 显示课程的完整信息
4. 用户点击返回按钮 → 回到课表主屏幕

### 流程 4: 登出
1. 用户进入设置屏幕
2. 点击登出按钮
3. 清除本地会话数据
4. 返回登录屏幕

## 颜色方案

该应用采用现代化的蓝色主题，符合浙江大学的品牌色调：

| 颜色用途 | 十六进制值 | 使用场景 |
|---------|----------|--------|
| **主色** | `#0a7ea4` | 按钮、标题、强调元素 |
| **背景色** | `#ffffff` (亮) / `#151718` (暗) | 屏幕背景 |
| **表面色** | `#f5f5f5` (亮) / `#1e2022` (暗) | 卡片、输入框 |
| **文本色** | `#11181C` (亮) / `#ECEDEE` (暗) | 主要文本 |
| **辅助文本** | `#687076` (亮) / `#9BA1A6` (暗) | 次要文本、描述 |
| **课程卡片** | 多色 | 不同课程用不同颜色区分 |

## 课表展示设计

### 表格结构
- **行**: 上课时间段（第1-2节、第3-4节等，共5-6个时间段）
- **列**: 星期一至星期日（7列）
- **单元格**: 课程卡片或空白

### 课程卡片设计
- **卡片尺寸**: 自适应，填充对应时间段
- **内容**: 
  - 课程名称（粗体，主要文本）
  - 教室位置（较小字体，辅助文本）
  - 节次信息（可选）
- **样式**:
  - 圆角边框
  - 阴影效果
  - 半透明背景
  - 不同课程用不同背景色

### 交互设计
- **点击反馈**: 卡片按下时缩放和透明度变化
- **长按**: 可能显示快速操作菜单（如添加提醒）
- **滑动**: 左右滑动切换周次

---

## 设计原则

1. **简洁性**: 界面清晰，信息层级明确
2. **可读性**: 字体大小和对比度满足可访问性标准
3. **响应性**: 适配不同屏幕尺寸的移动设备
4. **一致性**: 遵循iOS设计规范，提供原生应用体验
5. **性能**: 快速加载和流畅的交互
6. **可用性**: 直观的导航和操作流程

---

## 技术实现框架

- **框架**: React Native (Expo)
- **样式**: NativeWind (Tailwind CSS)
- **状态管理**: React Context + AsyncStorage
- **后端集成**: 通过脚本与浙江大学CAS系统通信
- **数据存储**: 本地AsyncStorage缓存课表数据
